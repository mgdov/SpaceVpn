# Тестовый пользователь

Для локального тестирования и разработки добавлен тестовый пользователь с моковыми данными.

## Данные для входа

- **Логин:** `test`
- **Пароль:** `test123`

## Доступные функции

При входе под тестовым пользователем вы получаете доступ к полностью функциональному личному кабинету с моковыми данными:

### Профиль пользователя

- **ID:** 999
- **Username:** test
- **Email:** test@spacevpn.com
- **Полное имя:** Тестовый Пользователь

### VPN Статус

- **Статус:** Активен
- **Тариф:** Тестовый тариф
- **Срок действия:** +30 дней от текущей даты
- **Использованный трафик:** 5 GB
- **Лимит трафика:** 100 GB (5%)

### VPN Ключи

- Доступен тестовый VPN ключ
- **Имя:** Мой тестовый ключ
- **UUID:** test-uuid-1234
- **VLESS ссылка:** vless://test-uuid-1234@example.spacevpn.com:443?type=tcp&security=reality&flow=xtls-rprx-vision#SpaceVPN-Test

### Тарифы

Доступны моковые тарифы для тестирования:

1. **Бесплатный** (0₽, 10 GB, 1 устройство)
2. **Стандарт** (299₽, безлимит, 3 устройства)

## Реализация

Тестовый пользователь работает полностью локально без вызова backend API:

1. **Аутентификация** - в `lib/auth-context.tsx` добавлена проверка на test/test123
2. **Токен** - сохраняется как `test_token_12345` в localStorage
3. **Данные профиля** - хранятся локально, не синхронизируются с backend
4. **VPN данные** - моковые данные в следующих компонентах:
   - `/app/account/page.tsx` - основная страница аккаунта
   - `/app/account/keys/page.tsx` - страница VPN ключей
   - `/app/account/tariffs/page.tsx` - страница тарифов

## Использование

1. Откройте страницу логина: http://localhost:3000/login
2. Введите логин `test` и пароль `test123`
3. Нажмите "Войти"
4. Вы будете перенаправлены на страницу личного кабинета с моковыми данными

## Примечания

- Тестовый пользователь работает только на фронтенде
- Операции, требующие backend (создание ключей, покупка тарифов), не будут работать
- Для полного тестирования используйте реальный backend API
- Тестовые данные сбрасываются при выходе из системы
